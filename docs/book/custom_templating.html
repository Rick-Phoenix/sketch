Custom Templating

Sketch can be used to leverage the Tera templating engine to quickly set up all sorts of files or entire project structures.

This means being able to inject variables into generated content, as well as using all of the functions and filters available in Tera (like extracting envs), plus extra ones added by Sketch (like uuid generation).

Templates can be defined in files or directly within a command. The variables can also be defined in the config file or as part of a command.

root_dir = "../output/custom_templates"

global_templates_vars = { my_var = 15 }

# We can define templates directly in the config file
templates = { lit_template = "my_var: {{ my_var }}" }

# Or as files inside a directory
templates_dir = "../templates"

Rendering templates

Templates can be rendered as a group, or individually.

This is how groups are defined:

# A group of templates that will be rendered together
templating_presets.test = [
  # Full path to the template file from templates_dir
  { template = "subdir/nested.j2", output = "new_dir/nested.yaml" },

  # Only using the global variables
  { template = "abc.j2", output = "with_global_var.yaml" },

  # Using a local override
  { template = "abc.j2", output = "with_override.yaml", context = { my_var = 20 } },

  # Defining a literal template
  { template = { name = "literal_inlined", content = "my_var: {{ my_var }}" }, output = "literal_inlined.yaml" },

  # Using a literal template defined in the config file
  { template = "lit_template", output = "literal_from_config.yaml" },
]

When we run the command

sketch render-preset test

The output of the specified root-dir will look like this:

.
├── from_literal.txt
├── literal_from_config.yaml
├── literal_inlined.yaml
├── new_dir
│   └── nested.yaml
├── with_cli_override.yaml
├── with_global_var.yaml
└── with_override.yaml

2 directories, 7 files

Global and Local Context

As you can see, the variables provided as a local context override global variables:

my_var: 20

But the variables defined from the command always have the greatest priority.

So if we run this command:

sketch --set my_var=25 render --id lit_template with_cli_override.yaml

The output will be:

my_var: 25
Variables defined with the --set flag must be formatted in valid json. This means that, for example, strings must be wrapped in escaped quotes.

Defining a template in the command

We can also define a template inside the command itself.

Cmd:

sketch --set location="Isengard" render --content "they're taking the hobbits to {{ location }}!" from_literal.txt

Output:

they're taking the hobbits to Isengard!

Rendering to stdout

Templates can also be rendered to stdout with the --stdout flag.
